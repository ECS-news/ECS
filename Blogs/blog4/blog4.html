Javascript: linking to HTML

Javascript DOMS

In order for javascript to be able to interact with HTML, a new file is created called a DOM which takes the existing HTML and converts it into a file in which javascript can interpret and use. This file replaces tags like

<h1> and <p>

with objects that are more useful for javascript. The data is in this file can then be accessed using global objects in javascript which we will talk more about further down this blog.

The two most notable global objects are the window object and document object

Window

The

window

object is at the top the hierarchy for the website and represents the browser in which the webpage is opened. The

window

object is usually omitted when calling particular methods and also could be used before

document

when calling the methods of

document

- more on that later.

  • window.innerWidth - returns the inner width of the browser (not including scrollbars and toolbars)


  • window.innerHeight - returns the inner height of the browser (not including scrollbars and toolbars)


  • window.outerWidth - returns the entire width of the browser including scrollbars and toolbars.


  • window.outerHeight - returns the entire height of the browser including scrollbars and toolbars


  • window.alert() - displays an alert box on the current window

  • window.confirm() - displays a box asking for confirmation from the user. Boolean return based on user's response

  • window.print() - prints the contents to the current window.


  • window.prompt - prompts the user for an input via a box

  • window.scrollTo() - scrolls to a specific coordinate on the screen

Document

The document object is apart of the window object but again can be ommited when calling methods for the document and refers to the actual content that is on the webpage itslef. the document object is very useful for being able to refer to particular elements and change the formatting of them

There is a whole list of different methods, the most important ones for finding elements shown below:

finding elements

  1. document.getElementById(id) - finds the element with the matching id.

  2. document.getElementByTagName(name) - finds elements by tag name.

  3. document.getElementByClassName(name) - finds elements with matching class name.

  4. document.querySelector(Selector - finds the first element on the page with the matching CSS selector (button, img etc)

now that we have found the element(s) we are looking for we can now change differrent properties of the element like its height or text.

Changing properties of elements

Syntax Result
element.innerHTML = "text" Changes the text in the element to the desired new text
element.attribute = val Changes the value in the attribute of an element to the new value
element.style.property = style Changes the style of an HTML element to a new style
element.setAttribute(attribute,value) = "text" Change the attribute value of an HTML element

Events

Events are an important feature of writing javascript code that is going to be interacting with the webpages

event handlers/listeners

The best way to create event listeners is via the addEventListener(event-type, listener)



event-type : this must be a string of the corresponding event you want to be the trigger for the Listener to respond to like “click” or “hover”.

another way you can create event handlers is via inline event handlers. To demonstrate their capabilities I have used them to create some of the buttons on this webpage excluding any which refer to height/width.

Note that here I have directly mixed HTML and javascript which is awful for maintaining the code. I could have made functions which did the neccessary code and called those in the HTML instead of writing the entire function but this is also a bad idea.



Inline handlers I would strongly not recommend for larger projects as they are deemed bad practice. A good habit of website building is keeping HTML, CSS and Javascript separate from one another. Inline event handlers are created in the HTML and were the first way websites could implement event handlers. For the purpose of this article, it is likely that inline event handlers won’t cause any problems as this article will not need much maintaining in the future and I am only using 5 buttons with this method but if I was to scale this webpage and have let's say a 100 buttons, it would quickly become a nightmare having to maintain 100 different buttons that refer to 100 or so functions located in a different file. We also lose out on all the benefits of addEventListener since we cannot iterate over a series of elements to apply the same event handler to all of them at once nor can we give multiple event handlers to one event on one element.

To finish off event listeners, the third and final way you can create them is via event properties. For example, buttons have the event property on followed by the event which you can assign an event handler. This is effectively the same as addEventListener but you cannot have multiple event handlers for one event.

This feature could be useful if you do want to overwrite event handlers however, the same result can be achieved using removeEventListener and addEventListener.
