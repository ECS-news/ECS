Javascript basics

Today I wanted to talk about javascript and all the different ways in which the language can be used in general. This blog will go into detail on the basics of javascript and its syntax. This blog will assume you have some coding knowledge such as this not being the first programming language you have learnt before.

What is javascript used for?

Javascript is used for web based applications but can be used standalone. Javascript can control and change different attributes and properties of HTML elements via referencing elements on a web page such as through their ID. Below will be a series of javascript basics followed by javascript specific keywords and concepts.

How to run javascript on web pages

This can be done by either writing the javascript in a <script> tag in the HTML file or by creating a separate javascript file and linking them by this line of code in the body of the html file.

<script src="(filename.js)"> </script>

Variables

Variables don’t need to be declared with a type like int or str but you do need to specify what “kind” of variable you are defining:

let (variable name) = (contents) ;
var (variable name)= (contents);
const (variable name) = (contents) ;

“Const” creates a fixed value known as a literal. Neither the name or the contents can be altered after declaring a constant.

“Let” and “var” create variables and ,as a result, can be assigned different values as the programme is running. The difference between “let” and “var” is from the scope in which they are defined. “Let” defines the variable in the current block and can only be used within that block. For example, if a variable was defined using “let” in an “if statement”, the variable could not be referenced outside the”if statement” as the variable would not be defined for that scope of the program. “Var” has a larger scope but not global scope. “Var” defined variables can be used throughout an entire subroutine of the program of where they are defined. If declared in main, they can be used throughout the entirety of main but not in subroutines. Note: the value can be used in subroutines by passing it as a parameter. Similarly, if they are declared in a subroutine, they can be referenced throughout the entire subroutine but not in the main program. Again, You can get around this by using a function and a return statement to pass the value to main.

Strings

Strings are defined using either “” or ‘’ or ``. There isn’t a difference between using the different ways of defining strings however, for ease of use I’d recommend using one type of way of defining them in order to make the code easier to read and understand which therefore also makes it more maintainable.

var string = “Hello World” ;
var string = ‘Hello World‘ ;
var string = `Hello World` ;

Note: the character data structure doesn’t exist in javascript. Characters will just be defined as strings of length one.

  • string.length () - returns the length of the string
  • string .substring(start,end) - returns substring between indexes start and end
  • see more at W3schools

Arrays

arrays are defined as shown below

const (arrayName) = [element1, element2 , element3…] ;

  • Arrays can have different variable types inside them such as other arrays and objects
  • Arrays are passed by reference
  • Arrays are indexed at 0 and so each element can be accessed via

    array[index]

  • array.length() - returns the number of elements in the array
  • arrayName.sort() - sorts the array

Other

var decimal = 5.5 ;
var decision = true ;
var decision = false ;

In terms of good variable names for javascript, developers tend to use either underscores to distinguish between different words in a variable name or they utilise lower camel case whereby the variable name is in all lowercase except for the first letter of words excluding the first letter e.g variableName.

Note: you cannot use hyphens in variable names as the hyphen symbol is reserved for subtraction.

Operations

  • (+) adding numbers or concatenating strings
  • (-) subtracting numbers
  • (*) multiplication
  • (/) Division
  • (%) Modulus division
  • No symbol for integer division
  • (**) exponents

Selection

  • (<) is less than
  • (>) is greater than
  • (*) multiplication
  • (!) Not
  • (==) is equal to
  • (>=) is greater than or equal to
  • (<=) is less than or equal to
  • (!=) not equal to

if statements

if ((statement1)) {
  // code for if statement1 is true
}
else if ((statement2)) {
 // code block for if statement 1 is false but statement 2 is true
}

else if ((statement3)) {
 // code block for if statement 1 is false and statement 2 is false but statement 3 is true
}
    …
    …
    …
else {
  //code block for if all statements before this were false
}

Example

var grade = 2 ;

if (grade == 3) {
    console.log("Well done! Great Score!") ;
}
else if (grade == 2) {
    console.log("Well done! You Passed!") ;
}
else {
    console.log ("You failed! Better luck next time") ;
}

Output

Well done! You passed

Switch case statements

switch(expression) {
  case x :
    break;
    // code block for if expression = x is true
  case y :
    break;
    // code block for if expression = x is false but expression = y is true
  case z :
    break ;
    // code block for if expression = x is false and expression = y is false but expression = z is true
    …
    …
    …

  default ;
    //codefor if all statements before this were false
}

Example:

var grade = 1 ;
switch(grade) {
  case 3 :
    break;
    console.log("Well done! Great Score!") ;
  case 2 :
    break;
    console.log("Well done! You Passed!") ;

    console.log ("You failed! Better luck next time") ;

Output

You can do this by replacing the while with do and moving the while statement to the line starting with } and ending the loop with a ;

Subroutines

All functions in javascript are methods of an object. If this isn’t specified like in the examples below, the function is a method of the global object window but the window can be omitted when the methods are called.

Syntax

function (subroutineName) (parameter(s)) {
// code block to be run when the function is called
}
subroutine(arguments(s))

If the subroutine is defined as a function then it must return a value or series of values to the main program using the return keyword.You can also create anonymous functions which don’t have a name like this:

const (variableName) = function(parameter(s)) {return: val } ;

Self-invoking

Note: If you add () after the contents of the function expression, when you declare the function, it will also call the function

let x = function (a,b) {return a * b} (6,5) ;
console.log(x) // outputs 30;

Arrow functions

Arrow functions allow a short-form way of writing function expressions

Const variableName = (parameter(s)) => {return: val} ;

Note: The return and {} can be omitted from arrow functions if the function is only one statement.
